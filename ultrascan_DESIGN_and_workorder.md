## 0. 役割（最初に読むこと）

あなた（Claude Code）は、Dodotronic UltraMic 250K（最大250kHzサンプリングの超音波マイク）から
**連続リアルタイムで超音波を可聴化・可視化・記録する**デスクトップアプリを、ゼロから段階的に実装する。
最終形は「バット検出器 + スペクトラムアナライザ + イベント録音機」。

担当者（Kali）は SDR 信号識別システム **sigscan**（HackRF + STFT→CNN）を別途開発している。
本プロジェクトは将来 sigscan の CNN 学習ハーネスを流用するため、**スペクトログラム出力の形を sigscan と揃える**。

## 1. 背景・確定した前提

| 項目 | 確定値 |
|---|---|
| 主モード | **連続ライブ監視（continuous live monitoring）を最優先**。スナップショット再生は後回し |
| OS | **Windows** |

| 取込API | **WASAPI 排他モード必須**（共有モードは禁止＝後述） |
| ヘテロダイン方式 | **複素ミックス（解析信号）＝鏡像なし**で最初から作る。実数cosミックス禁止 |
| 言語/環境 | Python 3.9 系（sigscan と揃える。CNN 流用のため torch 2.8.0+cpu / numpy 1.26.4 と互換に） |
| 描画 | pyqtgraph（matplotlib はリアルタイム不可） |

### 設計思想（sigscan から継承・確定）

1. **真実の源は1つ**：フルレート生サンプルの単一リングバッファ。表示・可聴化・録音はすべてここから読む。
2. **2パスを物理的に分離**：表示はフレーム落ちOK（最新優先）／音声は落とすとクリックになる（厳密FIFO）。処理を共有しない。
3. **Sim-first**：実機（マイク）の前に、**WAV ファイル or 合成信号**で全DSP経路を通す経路を必ず用意。
4. **画像で確認してから次へ**：表示変更後は既存ツール（例: Audacity / SDR系ウォーターフォール）と目視比較。
5. **量より質**：将来のCNN学習で、低品質・誤ラベルデータを土台に混ぜない。怪しいイベントは隔離。
6. **最小実装**：各マイルストーンはスコープを膨らませない。動くものを残してから次へ。

## 2. 最重要原則（絶対厳守）

### スレッドの鉄則
- **入力コールバックはコピーのみ**。FFT・GUI操作・重い処理を一切入れない（入れると音が途切れる）。
- **出力コールバックは SPSC キューからドレインのみ**。
- **Qt ウィジェットには GUI スレッド以外から触らない**。スレッド間受け渡しはキュー or Qt シグナル。
- numpy の重い処理は GIL を解放するので、DSP ワーカーの FFT 中も音声スレッドは回る（粗いロックを全体にかけない）。

### Windows / 取込の鉄則
- **WASAPI 排他モード**（`sounddevice` の `WasapiSettings(exclusive=True)`）で開く。
  共有モードはシステムミックスレート（≈48kHz）へ勝手にリサンプルし、**24kHz超が消える**＝ゴミになる。
  代替: ASIO ドライバ / WDM-KS。
- デバイスの**実ネイティブレートを起動時に検証**（250k で開けているか。ナイキスト125k）。
- 出力コールバックの**アンダーラン回避**：処理済み音声を先読みリングに積んでおく。

### PowerShell / コンソールの制約（sigscan と同じ）
- `&&` 不可。**コマンドは1行ずつ**実行。
- コンソールは cp932。**各 CLI は UTF-8 出力を強制**すること。
- `--db nul` 的な NUL デバイス指定は不可。一時ファイルを使う。

## 3. アーキテクチャ

```
マイク(250k, real)
   │
   ▼
[L0 取込スレッド]  ← WASAPI排他。コールバックは「生サンプルをコピーするだけ」
   │  copy
   ├───────────────► [L1 リングバッファ：生float32 / 数秒 / 真実の源]
   │                          │ 各コンシューマは自前の読み出しポインタを持つ（ポインタ非共有）
   │                          │
   │             ┌────────────┴───────────────┐
   │             ▼                             ▼
   │   [L2 DSPワーカー]                 [L3 可聴化ワーカー]
   │   rfft / STFT                      DDC鎖（下記）→ GainStage → soxr(→48k)
   │   → 表示フレーム                    → [出力リング(SPSC)]
   │       │                                   │
   │       ▼                                   ▼
   │   [表示キュー(最新優先・欠落OK)]      [L0' 出力コールバック：ドレインのみ]→ スピーカー
   │       │
   │       ▼
   └─► [L4 GUI: pyqtgraph]  ライブスペクトル + ウォーターフォール
            ▲  ドラッグで帯域選択 → L3 の LO/帯域幅へ
            │
       [L5 検出/測定/録音]  ← L1 と L2 を購読（v1後半）
```

### 各層の責務
- **L0 取込**：唯一のリスク層。生サンプルのコピーのみ。Xrun ステータスを監視・記録。
- **L1 リングバッファ**：取込/表示/再生のレートを完全分離する心臓。単一プロデューサ＋複数コンシューマ。
- **L2 DSP（表示用）**：実信号なので `rfft`、片側スペクトル 0〜125kHz。STFT 連続でウォーターフォール列を生成。
- **L3 可聴化**：連続モードの主役。DDC鎖で帯域を可聴域へ。専用の出力コールバック＋SPSCキューを持つ。
- **L4 GUI**：pyqtgraph。波形上を**ドラッグで帯域を囲む**→ それが可聴化の入力。
- **L5 検出/測定/録音**：イベント検出・パラメータ測定・GUANO録音（v1後半 M4/M5）。

## 5. DSP 詳細：DDC 鎖（鏡像なしヘテロダイン）

入力 `s[n]`（実数, fs=250kHz）、選択帯域 `[f_lo_sel, f_hi_sel]`、帯域幅 `W = f_hi_sel - f_lo_sel`。

```
1) 複素NCO:    lo[n] = exp(-j·2π·f_lo_sel·n / fs)        # 位相はブロック跨ぎで累積保持
2) ミックス:    x[n]  = s[n] · lo[n]                       # 選択帯域下端をDCへ。複素ベースバンド
3) LPF:        y[n]  = lowpass(x[n], cutoff = min(W, fs_dec/2))  # FIR。状態(zi)をブロック跨ぎ保持
4) デシメーション: 250k → 50k（÷5・整数）。LPFがアンチエイリアスも兼ねる
5) 実数化:      audio[n] = real(y_decim[n])               # 選択帯域が 0..W の可聴音になる
6) soxr:       50k → 48k（端数のみ高品質リサンプル）
```

- **鏡像除去の根拠**：複素指数 `exp(-jωn)` でミックス＋LPF だと、鏡像成分は `-2·f_lo_sel` 付近へ飛び LPF で消える。
  `cos(ωn)` 単独ミックスは上下両側へシフトして鏡像が折り返すため **禁止**。
- **連続性**：NCO 位相累積・FIR 状態(`scipy.signal.lfilter` の `zi`)を必ずブロック間で持ち越す。途切れるとプチノイズ。
- **自動ヘテロダイン追尾（M4 オプション）**：Detector が出した `f_peak` を `f_lo_sel` に自動設定し、
  鳴き声を可聴域中央へ自動配置（ダイヤル不要）。

## 6. マイルストーン（各段で必ず動くものを残す）

- **完了報告**：到達レイテンシ・許容バッファ・Xrun 結果を `M0_capture_report.md` に記録。

### M1 — 表示のみ
- [ ] L0→L1→L2→L4。ライブスペクトル + 横スクロールウォーターフォール（音声なし）。

- [ ] 検証：既存ツールとウォーターフォールの見た目を目視比較。
- 主要パラメータ：FFTサイズ / ホップ / 窓（周波数分解能 ↔ 時間分解能のトレードオフ）。

### M2 — 連続ヘテロダイン可聴化（連続モードの肝）
- [ ] `HeterodyneAudifier`（§5 の DDC 鎖、鏡像なし）。
- [ ] 専用**出力コールバック + SPSC キュー**。NCO位相・FIR状態の持ち越し。
- [ ] ウォーターフォール上を**ドラッグで帯域選択** → `f_lo_sel`/`W` へマップ → ライブ音声。
- [ ] 検証：既知の超音波源（鍵束のジャラ音、TVリモコンは赤外なので不可→**超音波式リモコン/動体センサ/コウモリ**）で可聴化確認。

### M3 — ゲイン / AGC（小音の拡大）
- [ ] `GainStage` プロトコル。`AGCGain`（連続既定）→ `NormalizeGain` → `CompressorGain`。
- [ ] アタック/リリース時定数を UI 露出。
- [ ] **制約をUI/コメントに明記**：ゲインを上げるとノイズフロアも上がる。ノイズに埋もれた信号は

### M4 — イベント検出 + 測定
- [ ] `Detector`（SNR/エネルギーベース）。イベント検出・セグメント化。
- [ ] 自動測定：f_peak / f_min / f_max / bandwidth / slope / duration / IPI。スペクトログラムにオーバーレイ。
- [ ] （オプション）自動ヘテロダイン追尾。

### M5 — イベント連動録音（GUANO）
- [ ] **プリトリガ・リングバッファ**：イベント開始前の数百msも保存（onset を取り逃さない）。
- [ ] イベント時のみ高レート WAV 書き出し。`write_event_wav`（guano-py でメタ埋め込み）。
- [ ] 怪しいイベントは隔離フォルダ（例: `captures/_review_pending/`）へ（sigscan の「量より質」）。

## 7. v2 以降（継ぎ目だけ v1 で用意しておく）

- **CNN 分類**：検出イベント → `spec_audio.render`（[256,256]）→ **sigscan と同じ CNN/学習ハーネスを流用**。
  ⚠ **学習データは未収集**。インフラを今作り、学習は後。ルール検出器の出力を教師にしない。
  ラベル体系は将来 sigscan に倣い「種/カテゴリ × 確信度」、確信度 < 0.7 は Unknown。
- **タイムエクスパンション audifier**：忠実再生（周波数比 1/N）。検出イベントを自動でタイムストレッチ再生。
- **超音波バンドプラン**：既知のコウモリ/昆虫/工業ノイズ帯を `Band(name, f_lo, f_hi, priority, hint)` で定義し、
  band-focus / ラベリングに使う（sigscan の 31バンド・帯域フォーカスと同型）。
- **GPS + GSI 地図連携**：検出を Kali の既存 GSI マップアプリへプロット（field 運用時）。
- **測定カーソル**：2点クリックで Δf / Δt / 傾きを読む。
- **ノイズ低減**：スペクトル減算 / ノイズゲート（アーティファクト注意）。
- **dB SPL 校正**：マイク校正データ必須。後回し。

## 9. テスト / 検証

- **Sim-first 単体テスト**：合成信号（既知の周波数のトーン/チャープ）を `spec_audio.render`・DDC・Detector に通し、
  数値で検証（実機不要）。例：45kHz トーン → ヘテロダイン後に期待の可聴周波数が出るか。
- **鏡像テスト**：単一トーンを入れ、鏡像（折り返し）が出ないことを確認（複素ミックスの正しさの担保）。
- **連続性テスト**：ブロック境界でプチノイズ（不連続）が出ないこと（NCO位相・FIR状態の持ち越し検証）。
- **目視検証**：ウォーターフォール画像を保存し、既存ツールと比較（「画像で確認してから次へ」）。
- 既存テストは無変更のまま全緑を維持。

## 11. 禁止事項

- 入力/出力コールバック内での **FFT・GUI操作・重い処理**。
- **凍結契約のシグネチャ変更**（人間承認なし）。
- **実数 cos 単独ミックス**（鏡像が折り返す）。必ず複素ミックス＋LPF。
- **WASAPI 共有モードでの取込**（24kHz超が消える）。
- **ルール/ヒューリスティック検出器の出力を CNN 教師に流用**（ラベルノイズ）。
- **スコープ膨張**。各 M は最小実装に留める。
- PowerShell で `&&` 連結、cp932 のまま UTF-8 出力を怠ること。
